# 📅 AI-помощник для Google Calendar

## 📖 Описание

Интеллектуальная система управления календарем через естественный язык с AI-помощником на базе **Qwen 3.5 (9B)**.

---

## ✨ Возможности

### 🗣️ Управление естественным языком

Вы можете общаться с помощником как с живым ассистентом:

```
📝 Примеры запросов:

• "Запиши меня к стоматологу завтра в 15:00"
• "Создай встречу с командой в понедельник в 10 утра"
• "Напомни о дне рождения мамы 15 мая"
• "Добавь тренировку по вторникам и четвергам в 19:00"
• "Перенеси встречу с Джоном на час позже"
• "Покажи свободное время на этой неделе"
• "Удали встречу с бюджетом"
```

### 🔒 Безопасное создание событий

**Все события создаются только после вашего подтверждения!**

```
Процесс создания:

1️⃣  Вы говорите: "Запиши меня к врачу завтра в 14:00"

2️⃣  AI распознаёт намерение и создаёт черновик:
    📋 Черновик события:
    • Название: Визит к врачу
    • Дата: 15.04.2026
    • Время: 14:00 - 15:00
    • Описание: Плановый визит к врачу

3️⃣  AI показывает черновик и ждёт подтверждения:
    ✅ Подтвердить создание события?
    Напишите "подтвердить" или "отмена"

4️⃣  Вы подтверждаете: "подтвердить"

5️⃣  Событие создаётся в Google Calendar ✨
```

### 📊 Контекстная осведомлённость

AI видит всё ваше расписание:

- ✅ **Google Calendar** — встречи, события, напоминания
- ✅ **Skyeng** — уроки, домашние задания, тесты, дедлайны, оценки
- ✅ **Дедлайны** — просроченные и предстоящие задачи
- ✅ **Свободное время** — анализ занятых слотов

```
Пример диалога:

Вы: "Хочу записать тренировку на завтра"

AI: "Завтра в 18:00 у вас урок Skyeng, а в 20:00 
     встреча в календаре. Предлагаю свободные слоты:
     
     • 14:00 - 16:00 (2 часа)
     • 19:00 - 20:00 (1 час)
     
     Какое время предпочитаете?"
```

```
👤 Вы: Запиши меня к стоматологу завтра в 15:00

• "изменить [поле]" — изменить параметр
```

### Сценарий 2: Поиск свободного времени

```
👤 Вы: Когда я могу сходить в спортзал на этой неделе?

🤖 AI:
📊 **Анализ расписания на неделю**

**Занятые слоты:**
• Пн 18:00-19:00 — Урок Skyeng
• Ср 14:00-15:00 — Встреча с командой
• Пт 19:00-20:00 — Урок Skyeng

**✅ Свободные слоты для тренировки:**

1. **Пн 10:00-12:00** (2 часа) ⭐ Рекомендуется
2. **Вт 16:00-18:00** (2 часа)
3. **Чт 09:00-11:00** (2 часа)
4. **Пт 12:00-14:00** (2 часа)

Записать тренировку на какое время?
```

### Сценарий 3: Изменение события

```
👤 Вы: Перенеси встречу с Джоном на час позже

🤖 AI:
📋 **Изменение события**

**Найдено событие:**
• Встреча с Джоном
• Текущее время: Ср 15.04 14:00-15:00

**Новое время:**
• Ср 15.04 15:00-16:00

✅ **Подтвердить перенос?**
```

### Сценарий 4: Удаление события

```

Напишите:
• "удалить" — подтвердить удаление
• "отмена" — отменить
```

### Распознавание намерений (Intent Recognition)

AI использует продвинутый NLP для распознавания:

| Тип намерения | Примеры триггеров |
|--------------|-------------------|
| `create_event` | "запиши", "создай", "добавь", "напомни" |
| `update_event` | "перенеси", "измени", "обнови" |
| `delete_event` | "удали", "отмени", "убери" |
| `find_free_time` | "когда я могу", "найди свободное" |
| `list_events` | "покажи события", "что у меня" |

## 🔐 Безопасность и конфиденциальность

### Защита данных

- ✅ **Локальное хранение** — все данные хранятся в вашей БД
- ✅ **Шифрование токенов** — OAuth токены зашифрованы
- ✅ **Подтверждение действий** — критичные операции требуют подтверждения
- ✅ **Сессионная изоляция** — каждый пользователь видит только свои данные

### OAuth 2.0 авторизация

```
1. Пользователь нажимает /login_google
2. Открывается страница авторизации Google
3. Пользователь предоставляет доступ к календарю
4. Google возвращает OAuth токен
5. Токен шифруется и сохраняется в БД
6. Access token обновляется автоматически
```

## ❓ FAQ

### Q: AI не создаёт событие, что делать?

**A:** Проверьте:
1. Подключён ли Google Calendar (`/status`)
2. Есть ли свободное время в указанном слоте
3. Правильно ли распознано намерение (смотрите логи)

### Q: Как отменить создание события?

**A:** Напишите "отмена" или "/cancel" в ответ на черновик события.

### Q: Можно ли создать повторяющееся событие?

**A:** Пока нет. Эта функция в разработке.

### Q: AI неправильно понял время, как исправить?

**A:** Уточните: "Нет, я имел в виду завтра в 15:00" или используйте явный формат "15.04.2026 15:00".

### Q: Где хранятся мои данные?

**A:** Все данные хранятся локально в PostgreSQL базе данных. Токены зашифрованы.
